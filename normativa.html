Introducción

En nuestra misión de construir un ecosistema digital verdaderamente inclusivo, la accesibilidad web es nuestra prioridad. Esta plataforma está diseñada bajo los principios de las Pautas de Accesibilidad al Contenido Web (WCAG), asegurando que personas con diversas discapacidades—visuales, auditivas, cognitivas o motoras—puedan percibir, entender, navegar e interactuar con el contenido. Creemos firmemente que la web es un derecho fundamental, y nos comprometemos a mantener un espacio donde la tecnología no sea una barrera, sino un puente para que todos los usuarios puedan acceder a la información y participar activamente.

Niveles de conformidad

Nivel Descripción Objetivo Uso
A Accesibilidad baja (mínimo) Satisfacer las necesidades más esenciales de accesibilidad para que el contenido sea posible de usar por algunas personas con discapacidad. Es el punto de partida y el mínimo legal en algunas jurisdicciones, aunque generalmente se considera insuficiente para una buena experiencia de usuario.
AA Accesibilidad Mejorada (Intermedio) Eliminar las barreras más comunes y significativas, haciendo el contenido usable y operable para la mayoría de las personas con discapacidad. Es el nivel más comúnmente adoptado y recomendado para sitios web y aplicaciones, y es un requisito legal frecuente en muchos países. Abarca criterios clave como el contraste de color adecuado y la navegación completa con teclado.
AAA Accesibilidad Excepcional (Máximo) Abordar las necesidades de accesibilidad más complejas, proporcionando una experiencia de usuario óptima para el mayor número posible de personas con discapacidad. Es difícil de alcanzar para sitios web completos, ya que algunos criterios AAA pueden requerir un diseño muy específico o no ser aplicables a todo tipo de contenido. A menudo se aplica a secciones específicas o contenido especializado.

Buenas prácticas

  1. Familiarice a su equipo con las WCAG, las últimas actualizaciones de las directrices y los requisitos de cumplimiento aplicables.
  2. Incluye etiquetas de texto alternativo para tus imágenes
  3. Haz que tu contenido sea fácil de leer y ver
  4. Habilite el control de audio para los usuarios de su sitio
  5. Proporcionar funcionalidad completa del sitio con el teclado
  6. Evitar el contenido intermitente
  7. Facilite la navegación por su sitio web
  8. Proporcionar mensajes de error útiles
  9. Utilizar código HTML bien escrito y compatible con las tecnologías de apoyo.
